Project Title: ConsultAfrique Dynamic Portal & Registration System Role: Senior Full-Stack Developer & System Architect

Objective: Build a comprehensive web platform for "ConsultAfrique" (Education & Medical Tourism). The system requires a public marketing frontend, a secure Client Portal (Student/Patient), and an Admin Dashboard.

1. Tech Stack & Architecture
Framework: Next.js (App Router) with React.

Styling: Tailwind CSS (Responsive, Mobile-First).

Backend/Auth: Supabase, Firebase, or Node.js (Must support Role-Based Access: User vs. Admin).

Transactional Email Service: SendGrid, Mailgun, or Nodemailer (SMTP).

Brand Identity:

Primary: Deep Green (Approx. #2F855A).

Accent: Orange (Approx. #ED8936).

2. Core Functional Flows
A. Two-Step Registration & Onboarding
Step 1: Sign Up (Low Friction)

User Action: Select "Student" or "Patient". Enter Email + Password.

System Action: Create account and send Email Verification link.

Step 2: Onboarding (After Email Verified)

Redirect: User is sent to the "Complete Profile" page on the Dashboard.

Required Fields: Full Name, DOB, Gender, Nationality, Address, Phone Number.

Missing Info Alert: If incomplete, show a persistent banner: "Please complete your profile and upload documents to proceed."

B. Dynamic Document Uploads (Dashboard)
Depending on the user type (Student vs. Patient), the "Documents" tab must request different files.

If STUDENT:

International Passport Copy (Min. 12 months validity) [Mandatory but can be skipped for later]

Academic Certificates (Primary, Junior Secondary, WAEC/NECO Senior Certificate - Original Results) [Mandatory]

Sponsorship Proof (Bank statements & Affidavit of Support)

Medical Certificate (Yellow Fever Vaccination) [Mandatory for Pakistan Visa]

If PATIENT:

International Passport Copy [Mandatory]

Referral Letter (From current physician)

Medical Reports & Diagnostics (Lab reports, Imaging CDs/USBs, MRI/CT scans, Biopsy reports) [Mandatory]

Current Prescriptions & Medication List

Medical History Form (Text fields for: Diagnosis, Symptoms, Past Treatments, Reason for choosing Pakistan)

Emergency Contact & Accompanying Person Passport Copy [Mandatory]

C. Email Sender & Notification Logic
Configure the SMTP/Mailer to use specific "From" addresses for outgoing client emails and "To" addresses for admin alerts:

1. Registration & Admissions (Students & Patients)

Sender Identity: admission@consultafrique.com

Use Case: Sending "Verify Email", "Welcome to ConsultAfrique", and "Action Required: Complete Profile" emails to users.

Admin Alert: When a new user registers, notify admins at this same address.

2. General Inquiries

Sender Identity: info@consultafrique.com

Use Case: Auto-acknowledgment for Contact Form submissions.

3. Support & Passwords

Sender Identity: support@consultafrique.com

Use Case: Sending "Reset Password" links or Technical Support replies.

4. System Critical/Admin

Recipient Only: admiiins@consultafrique.com

Use Case: Critical system errors or high-level admin notifications.
